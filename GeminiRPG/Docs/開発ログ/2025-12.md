↓
Geminiに聞いたところ、GitリポジトリをiCloudに入れるのは危険！⚠️ということなので以下の運用に変更
⚠️ 理由: Gitは .git という隠しフォルダの中に数千個の小さなファイルを管理しています。iCloudなどの自動同期サービスは、この大量のファイルの変更タイミングを正しく同期できず、**「PCでコミットしたのにスマホで壊れる（またはその逆）」**という事故が頻発します。リポジトリが壊れると、履歴が全部飛びます。

🛠 新・推奨運用フロー
1. マスタデータ: GitHubに置く（非公開リポジトリ）。

2. PCでの作業: VS Codeなどで普通に開発＆Push。

3. スマホでの作業:
• Working Copy アプリを開く。
• 編集したいCSV（例：Chapter_1.csv）を開いて、セリフを修正・追記。
• その場で「Commit & Push」。

4. AIとの連携:
• スマホでコードやCSVをコピーしたい時も、Working Copyから直接コピーできます。
これなら、「修正データを消し忘れる」ことも「iCloud同期のコンフリクトでファイルが壊れる」こともありません。

「Working Copy」、有料級の働きをしますが、基本機能は無料で使えるので一度試してみてください！

スマホでの作業

   → Chapter1.csv を編集
   → Commit: "[Chapter1] セリフ修正"
   → Push

3. アイテムデータを追加したい
   → Working Copy 起動
   → dev ブランチ
   → Items.csv を編集
   → Commit & Push

4. コピペして AI に読ませたい
   → Working Copy でファイルを開く
   → 選択 → コピー
   → Claude/Gemini に貼り付け

PCでの作業
1. スマホでの変更を取り込む
   $ git checkout dev
   $ git pull

2. Unity で動作確認
   - Play ボタン
   - エラーがないか確認

3. 問題なければ main にマージ
   $ git checkout main
   $ git merge dev
   $ git push

4. 大きな実装作業
   - 新機能ブランチを作る
   $ git checkout -b feature/battle-system
   - 実装
   - 動作確認
   - main にマージ
