### 概要
**シナリオCSVの大規模リファクタリング完了！** 16列の固定カラム方式から5列（Label, Command, Speaker, Text, Args）の汎用パラメータ方式に移行。視認性が大幅に向上し、シナリオの台本としての読みやすさを実現。Branchコマンドの仕様もJumpIf方式に変更し、バグの温床を解消。

### 作業1: CSVフォーマットの刷新
- **旧形式（16列固定）**
  ```csv
  Label,Command,Speaker,Text,Duration,FaceID,Position,Target,JumpTo,FlagName,ParamName,Value,BGM,SE,BgImage,Color
  ,FadeOut,,,2,,,,,,,,,White,,
  ```
- **新形式（5列 + Args）**
  ```csv
  Label,Command,Speaker,Text,Args
  ,FadeOut,,,time=2; color=white
  ```
- **メリット**
  - 空欄が激減し、CSVが台本として読める
  - Speaker/Textが独立しているので会話の流れが追いやすい
  - 新しいパラメータの追加が容易（Argsに追加するだけ）

### 作業2: スクリプトのリファクタリング
- **ScenarioCommand.cs**
  - 16個の個別フィールド → `Label`, `Command`, `Speaker`, `Text` + `Dictionary<string, string> Args`
  - アクセサメソッド追加: `GetString()`, `GetFloat()`, `GetInt()`, `GetBool()`, `HasArg()`
- **ScenarioLoader.cs**
  - 5列パース対応
  - Args列を `key=value; key=value` 形式で解析
  - フォーマットエラー時は警告を出してスキップ（停止しない）
- **ScenarioExecutor.cs**
  - 全コマンドを `cmd.GetString("key")` / `cmd.GetFloat("key")` 形式に変更
  - **Branch を JumpIf 方式に修正**: `flag=X; jumpTo=Y` でフラグがtrueならジャンプ
  - **空コマンドで continue するよう修正**: ラベル行で止まるバグを解消

### 作業3: 新コマンド追加
- **TextWait**: テキストをパッと表示して指定秒数後に消える（フェードなし）
  - 用途: 勢いのあるセリフ（「……待って！」など）
  - Args: `time=秒数`

### 作業4: CSV移行
- **Test_Dialogue.csv**: 16列 → 5列に変換
- **Chapter1.csv**: 16列 → 5列に変換、コメント行でセクション区切りも追加

### 学んだこと
- **CSVの視認性はシナリオ執筆の生産性に直結する**
  - ノベルゲームのデータの9割は「誰が」「何を喋ったか」
  - Speaker/Textを独立させることで台本として読める
- **データ量が少ないうちにリファクタリングすべき**
  - Chapter1.csv: 34行、Test_Dialogue.csv: 15行 → 移行コストが低い
  - データが膨大になってからでは手遅れ
- **Branch（条件分岐）の「次の行をスキップ」方式はバグの温床**
  - JumpIf方式（`flag=X; jumpTo=Y`）の方が直感的で安全

### 作業2: 立ち絵の描画品質・配置調整
- **アスペクト比の自動保持**
  - 問題: 正方形のImage枠に長方形のイラストを入れると変形してしまう
  - 解決: `ScenarioExecutor` 内で `targetImage.preserveAspect = true;` をコードから強制適用（Source Imageがnullの状態でも機能するように）
- **画質の改善（ジャギジャギ対策）**
  - 問題: 線がガビガビで汚く見える
  - 解決: Unityの画像インポート設定を変更
    - **Filter Mode**: Point（ドット絵用）→ **Bilinear**（イラスト用）に変更
    - **Compression**: High Quality（またはNone）に設定
    - **Generate Mip Maps**: オンにして縮小時の画質劣化を防止
- **配置の最適化**
  - RectTransformのPivotを「足元（X:0.5, Y:0）」に設定
  - Anchor（アンカー）を左下・右下・下中央に設定し、解像度が変わっても位置ズレしないように調整

- **Unityの画像設定**: 高解像度イラストを縮小表示する場合は `Generate Mip Maps` と `Bilinear` フィルターが必須。デフォルト設定はドット絵や3Dテクスチャ向けの場合がある。
- **堅牢なCSV読み込み**: Excel等で編集すると末尾の空列が消えることが多いため、コード側で列数不足を補完するロジックを入れておくのが安牌。
- **動的なUI設定**: `Preserve Aspect` などはInspectorだけでなく、コードで明示的にONにすることで事故を防げる。

  - 削除したメソッド: `GetCharacterImageByPosition()`
  - 約80行のコード削減

  - 問題: カイリ→ナギに話者が変わっても立ち絵が切り替わらない
  - 原因: `displayedCharacter != null` の場合、表情更新しかせず、別キャラへの切り替えロジックがなかった

### 学んだこと
- **式形式プロパティ（`=>`）と通常プロパティ（`get { }`）の使い分け**
  - 単純な計算：`public int MaxHP => baseHP + equipHP;`
  - 複雑な処理：`public int Attack { get { /* 複数行処理 */ } }`
- **foreach と配列の使い方**
  - `foreach (var emotion in emotions)` で配列を1つずつ処理

- **staticクラスの利点**
  - インスタンス化不要で `EmotionEffect.GetAttackModifier()` と直接呼び出せる
  - 計算ロジックのみを持つクラスに最適
- **Unityのファイル配置ルール**
  - `Assets/Scripts/`: コード（.cs）専用
  - `Assets/Resources/`: データファイル（.asset, CSV等）専用
  - `.asset` と `.meta` は常にペアで移動

## 20260103 土

ステータスシステムの設計を固めて、キャラクターデータを作成した日。ネフェシエルのステータスを参考に、尖った成長幅に。実際のプレイ感が気になる。

### やったこと
- StatusParameter.cs修正
  - baseEvasion削除（回避率はキャラ固有ではなく装備・バフで変動）
  - buffEvasion追加（恐怖やスキルによる一時的な回避バフ用）
- CharacterData.cs修正
  - 成長しないステータス（防御/抗魔/敏捷）のgrowth削除
  - 成長するのはHP/MP/攻撃/魔力の4つのみに
- 全キャラのCharacterData作成（6アセット）
  - ナギ、ミズキ、ペチ（猫）、ペチ（黒豹）、カグヤ、ハヤテ
- ステータス仕様書作成（Docs/データ類準備/設計書・仕様書/ステータス仕様書.md）

### 設計決定事項
- **成長するステータス**: HP, MP, 攻撃力, 魔力（ランクに応じた成長率。詰み防止のため）
- **固定ステータス**: 防御力, 抗魔力, 敏捷性（装備依存）
- **命中計算式**: `100 - (100 - スキル命中) × (相手敏捷 / 自敏捷 + 1) / 2 - 相手回避値`
- **回避値**: 装備回避 + バフ回避（キャラ固有の回避率は廃止）
- **ペチの変身**: CharacterDataを2つ持つ方式（猫/黒豹で別アセット）

### 学んだこと
- `public int MaxHP => baseHP + equipHP;` は式形式プロパティ。getterのみの読み取り専用で、呼ばれるたびに計算される
- GetStatusAtLevel()は初期化用のファクトリーメソッド。レベルアップ時ではなく、キャラ加入時やロード時に使う
