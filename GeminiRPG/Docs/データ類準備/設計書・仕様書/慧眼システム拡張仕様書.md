# 慧眼システム拡張仕様書

**作成日**: 2026-03-15
**ステータス**: 設計確定・実装待ち
**関連**: 戦闘システム（BattleManager）、敵データ（EnemyData）

---

## 1. 概要

慧眼（けいがん）は鏡のかけら収集で解放されるパッシブスキル（加護）であり、戦闘中に敵の弱点・耐性などの情報を表示する。

本仕様では、敵のランク（通常/色違い/ヌシ/ボス）に応じて情報開示の方式と速度を変えることで、「観察と推理の成功体験」を段階的に提供する設計とする。

### 設計思想

- 難易度は必要な知識量と相関する → 敵が強いほど情報獲得に手間がかかる
- ルールからパターンが生まれ、パターンの推理が成功した時が最も楽しい → 情報を段階開示することで推理の余地を残す
- 各キャラの尖ったステータスが慧眼と噛み合い、全員に役割が生まれる設計とする

---

## 2. 情報開示方式

敵ランクによって開示方式が異なる。

| 敵ランク | 開示方式 | 開示速度 |
|----------|----------|----------|
| 通常モンスター | 遭遇回数方式 | 速い（1〜2回） |
| 色違い個体 | 遭遇回数方式 | 普通（2〜4回） |
| ヌシ | 遭遇回数方式 | 遅い（3〜5回） |
| ボス | ターン経過方式 | 戦闘中に段階開示 |

---

## 3. 遭遇回数方式（通常・色違い・ヌシ）

同一種の敵と戦闘した累計回数に応じて、開示される情報が増える。撤退・敗北した戦闘もカウントに含む。

### 3a. 通常モンスター

| 遭遇回数 | 開示情報 |
|----------|----------|
| 1回目 | 敵名、種族、弱点属性（全て） |
| 2回目以降 | 耐性属性、状態異常耐性（全情報開示完了） |

- 雑魚敵はサクサク情報が集まる設計
- 序盤のプレイヤーに「慧眼を使って弱点を突く」基本サイクルを学習させる

### 3b. 色違い個体

| 遭遇回数 | 開示情報 |
|----------|----------|
| 1回目 | 敵名、種族、属性の傾向（「炎に関係がありそうだ」等のテキスト） |
| 2〜3回目 | 弱点属性が確定表示 |
| 4回目以降 | 耐性属性、状態異常耐性（全情報開示完了） |

- 通常種の知識が転用できるため、推測のヒントはある
- 確信を得るには実戦を重ねる必要がある

### 3c. ヌシ

| 遭遇回数 | 開示情報 |
|----------|----------|
| 1回目 | 敵名、種族のみ |
| 2〜3回目 | 属性の傾向テキスト |
| 3〜4回目 | 弱点属性が確定表示 |
| 5回目以降 | 耐性属性、状態異常耐性（全情報開示完了） |

- 倒すのが困難な敵のため、撤退→再挑戦のサイクルで情報が蓄積される
- 情報が揃った状態での攻略成功が大きな達成感を生む

---

## 4. ターン経過方式（ボス）

ボス戦では遭遇回数ではなく、戦闘中のターン経過に応じて情報が段階的に開示される。長く生き残るほど有利になる設計。

### 4a. 通常ボス（カグヤ戦、盗賊の親玉、ダンケル等）

| ターン数 | 開示情報 |
|----------|----------|
| 戦闘開始時 | 敵名、種族のみ |
| 3ターン経過 | 弱点属性が1つ開示 |
| 5ターン経過 | 残りの弱点属性＋耐性属性を開示 |
| 8ターン経過 | 状態異常耐性、特殊行動パターンのヒント |

### 4b. パートナーボス（カイリ戦、ルイ戦）

| ターン数 | 開示情報 |
|----------|----------|
| 戦闘開始時 | 敵名のみ（種族：人は表示） |
| 3ターン経過 | 弱点属性を開示 |
| 5ターン経過 | 耐性属性を開示 |
| 7ターン経過 | 使用スキルの属性傾向 |

- NPC時代に仲間だったキャラの弱点をプレイヤーは知らない
- 信頼していた相手が敵になるシナリオ上の衝撃と、戦闘知識がリセットされるゲームプレイ上の衝撃が一致する設計

### 4c. パートナーボス・鏡取り込み後（カイリ(鏡)、ルイ(鏡)）

| ターン数 | 開示情報 |
|----------|----------|
| 戦闘開始時 | 敵名のみ |
| 3ターン経過 | 「通常の弱点が通用しないようだ」というテキスト |
| 5ターン経過 | 耐性情報（全属性耐性であることが判明） |
| 8ターン経過 | 「時折、防御が緩む瞬間がある」というヒント |

- 鏡のかけらを取り込んだことで通常時とは異なる耐性を持つ
- 8ターン目のヒントが忘我状態（レフレクトス戦への伏線）の存在を示唆する

### 4d. レフレクトス（ラスボス）

| ターン数 | 開示情報 |
|----------|----------|
| 戦闘開始時 | 敵名、種族：鏡 |
| 3ターン経過 | 「ほぼ全ての攻撃が通用しない」（全属性耐性の示唆） |
| 5ターン経過 | 全属性耐性・全状態異常耐性を確定表示 |
| 8ターン経過 | 「数ターンおきに意識が揺らぐ瞬間がある」（忘我状態の開示） |
| 忘我発生ターン | 「今だ！防御が完全に消えている！」（忘我状態の視覚強調） |
| 10ターン経過 | 「猛毒だけはわずかに効く可能性がある」（猛毒低確率有効の開示） |

- プレイヤーがまず絶望し、次に突破口を見つけ、最後に攻略法を確立するという感情の流れを情報開示で設計する
- 小さなオルゴール使用時：忘我状態を任意に誘発できることが判明（アイテム知識とパターン分析の複合的な推理の報酬）

## 5. 戦術的シナジー

慧眼のターン経過方式は、各キャラクターのステータス特性と噛み合う設計とする。

| キャラ | 慧眼との関係 |
|--------|-------------|
| ナギ/ミズキ | 主人公としてバランスよく戦いながら情報蓄積を待つ |
| ペチ | 挑発ステップで敵の攻撃を引き受け、他メンバーの生存を確保。敏捷S＋高回避率で被弾を抑えつつターンを稼ぐ |
| カグヤ | 敏捷Dのため行動順が遅い → そのターンで新たに開示された情報を確認してから最適な魔法を選択できる |
| ハヤテ | 回復でパーティを延命する → 長期戦＝情報開示の促進。ハヤテの存在が慧眼の価値を最大化する |

## 6. データ設計

### 6a. EnemyDataへの追加フィールド

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `enemyRank` | enum (Normal / Rare / Boss_Normal / Boss_Partner / Boss_Mirror / Boss_Final) | 敵ランク |
| `insightStages` | List&lt;InsightStage&gt; | 情報開示段階の定義 |

### 6b. InsightStage構造体

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `threshold` | int | 開示に必要な遭遇回数 or ターン数 |
| `revealType` | enum (WeaknessPartial / WeaknessFull / ResistFull / AilmentResist / HintText / SpecialPattern) | 開示する情報の種類 |
| `hintText` | string | HintText時に表示するテキスト（例：「炎に関係がありそうだ」） |

### 6c. 戦闘マネージャーへの追加

| データ | 管理場所 | 説明 |
|--------|----------|------|
| 遭遇カウント | セーブデータ内（Dictionary&lt;enemyID, int&gt;） | 敵種別ごとの累計遭遇回数 |
| 現在ターン数 | BattleManager | 戦闘中のターンカウント（ボス用） |

## 7. UI表示仕様

### 7a. 慧眼未習得時

- 敵情報パネルは表示されない

### 7b. 慧眼習得後・情報未開示

- 敵情報パネルは表示される
- 未開示の項目は「???」で表示
- 「まだよくわからない」等のフレーバーテキスト

### 7c. 部分開示時

- 開示済みの弱点属性アイコンが表示される
- 未開示の項目は「???」のまま
- ヒントテキストがある場合はパネル下部に表示

### 7d. 全情報開示時

- 弱点属性アイコン、耐性属性アイコン、状態異常耐性が全て表示
- ボス戦の特殊パターンヒントも表示

### 7e. 新情報開示時の演出

- ターン経過で新情報が開示された瞬間にSE再生＋パネルのハイライト演出
- 「新たな情報を見抜いた！」等のシステムメッセージ表示
- さっきの面白さ数値で言えば、1回の開示につき **+5〜+10** の達成感を提供

## 8. バランス調整上の注意

- 通常モンスターの情報開示が遅すぎると序盤のテンポが悪化する。1〜2回で完了が適切
- ボス戦のターン経過開示が早すぎると推理の余地がなくなる。3ターン以上は耐える前提の設計
- ハヤテの回復が「情報獲得のための延命手段」として機能するため、ハヤテ不在時の難易度が跳ね上がる。これは意図的な設計だが、アイテムによる回復でもカバー可能であることを保証する
- レフレクトス戦の10ターン目の猛毒ヒントは、カグヤの「毒の力」加護（猛毒化）との連携を意識した配置。加護を取得していないプレイヤーにも攻略ルート（忘我状態を待つ）は保証する
