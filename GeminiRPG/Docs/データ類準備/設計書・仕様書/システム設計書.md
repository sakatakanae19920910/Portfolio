## ■ 主要なデザインパターン

### 1. Singleton パターン

```
SoundManager:
  - static Instance プロパティ
  - Awake() で唯一のインスタンスを保証
  - DontDestroyOnLoad で永続化
  - どこからでも SoundManager.Instance でアクセス可能
```

### 2. Command パターン（データ駆動）

```
ScenarioCommand:
  - CSV の 1行 = 1コマンド
  - 実行可能な命令をデータ化
  - ScenarioExecutor がコマンドを順次実行
```

### 3. Strategy パターン

```
ScenarioExecutor:
  - switch (command) で実行戦略を切り替え
  - 各コマンドごとに異なる Handler メソッド
  - HandleTextFade(), HandleBGM(), HandleChoice(), etc.
```

### 4. State パターン（暗黙的）

```
ScenarioExecutor:
  - currentIndex: 現在の実行位置
  - isWaiting: 選択肢待機中フラグ
  - flags: ストーリー分岐用フラグ管理
```

## ■ 重要な概念

### CSV → データ → 実行 の流れ

```
1. データ層:
   Chapter1.csv (人間が編集しやすい形式)

2. モデル層:
   ScenarioCommand (プログラムが扱いやすい形式)

3. 変換層:
   ScenarioLoader (CSV → ScenarioCommand)

4. 実行層:
   ScenarioExecutor (ScenarioCommand を解釈して実行)

5. 表示・音声層:
   UI (TextMeshProUGUI, Image, Button)
   Audio (AudioSource)
```

### シングルトンの永続化

```
BootScene:
  SoundManager 生成
    ↓ Awake()
  DontDestroyOnLoad(gameObject)
    ↓
  特殊シーン "DontDestroyOnLoad" に移動
    ↓
  シーンが切り替わっても消えない
    ↓
  すべてのシーンから SoundManager.Instance でアクセス可能
```
