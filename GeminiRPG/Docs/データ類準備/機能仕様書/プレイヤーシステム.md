# プレイヤーシステム機能仕様書

**最終更新**: 2026-01-25

---

## 概要

プレイヤーシステムは、マップ上でのプレイヤー移動、NPC会話、カメラ追従、マップ遷移、画面フェードを管理します。

---

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `PlayerController.cs` | プレイヤーの移動制御 |
| `NPCTrigger.cs` | NPC会話トリガー |
| `CameraFollow.cs` | カメラのプレイヤー追従 |
| `MapTransition.cs` | マップ間の遷移処理 |
| `FadeManager.cs` | 画面フェード（暗転・明転） |

---

## PlayerController.cs

### クラス概要

WASD / 矢印キー / ゲームパッドでプレイヤーを移動させる MonoBehaviour。

### Inspector参照

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `moveSpeed` | float | 移動速度（デフォルト: 5.0） |
| `inputActions` | InputActionAsset | Input System |
| `animator` | Animator | アニメーション（オプション） |

### 主要プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `CanMove` | bool | 移動可能フラグ（false で移動禁止） |

### 主要メソッド

| メソッド | 説明 |
|---------|------|
| `Teleport(Vector3 pos)` | 指定位置にワープ |
| `SetDirection(Vector2 dir)` | 向きを設定（ワープ後など） |

### 動作仕様

1. **入力**: Input System の `Player/Move` アクションを使用
2. **物理**: `Rigidbody2D.linearVelocity` で移動（壁との衝突は物理で処理）
3. **アニメーション**: Animator があれば `MoveX`, `MoveY`, `IsMoving` パラメータを更新

### 必須コンポーネント

- `Rigidbody2D`（自動追加）

## NPCTrigger.cs

### クラス概要

NPCにアタッチし、プレイヤーが近づいて Interact ボタンを押すとシナリオを開始する。

### Inspector参照

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `scenarioCSV` | TextAsset | 開始するシナリオCSV |
| `startLabel` | string | 開始ラベル（空なら先頭から） |
| `inputActions` | InputActionAsset | Input System |
| `interactIcon` | GameObject | 話しかけ可能アイコン（オプション） |

### 動作仕様

1. **検知**: `OnTriggerEnter2D` でプレイヤーが範囲内に入ったことを検知
2. **アイコン**: `interactIcon` が設定されていれば表示
3. **開始**: `Interact` ボタン（E キー長押し）で `ScenarioExecutor.StartScenario()` を呼び出し
4. **移動禁止**: 会話中は `PlayerController.CanMove = false`
5. **終了**: シナリオ終了時にコールバックで `CanMove = true` に戻す

### 必須コンポーネント

- `Collider2D`（自動追加、Is Trigger = true に自動設定）

### 使用手順

1. NPC用 GameObject を作成
2. `NPCTrigger` を追加（Collider2D が自動追加される）
3. Inspector で `scenarioCSV` と `inputActions` を設定

## CameraFollow.cs

### クラス概要

Main Camera にアタッチし、プレイヤーを滑らかに追従するカメラシステム。

### Inspector参照

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `target` | Transform | 追従するターゲット（通常はプレイヤー） |
| `autoFindPlayer` | bool | Playerタグで自動検索（デフォルト: true） |
| `smoothSpeed` | float | 追従速度（1〜20、デフォルト: 5） |
| `offset` | Vector3 | カメラ位置のオフセット（デフォルト: 0, 0, -10） |
| `useBoundary` | bool | カメラ移動範囲を制限するか |
| `boundaryMin` | Vector2 | 境界の最小座標 |
| `boundaryMax` | Vector2 | 境界の最大座標 |
| `useDeadZone` | bool | デッドゾーンを使用するか |
| `deadZoneSize` | Vector2 | デッドゾーンのサイズ |

### 主要メソッド

| メソッド | 説明 |
|---------|------|
| `SetPositionImmediate(Vector3 pos)` | カメラを即座に移動（マップ遷移時） |
| `SetTarget(Transform target)` | 追従ターゲットを変更 |
| `SetBoundary(Vector2 min, Vector2 max)` | カメラ境界を設定 |
| `DisableBoundary()` | カメラ境界を無効化 |

### 動作仕様

1. **追従**: `LateUpdate` でプレイヤー移動後にカメラを更新（ブレ防止）
2. **スムーズ移動**: `Vector3.Lerp` で滑らかに追従
3. **自動検出**: `autoFindPlayer = true` なら "Player" タグのオブジェクトを自動検索
4. **境界制限**: `useBoundary = true` でカメラが指定範囲外に出ないよう制限
5. **デッドゾーン**: `useDeadZone = true` でターゲットが範囲内にいる間はカメラが動かない

### Gizmo表示

- **緑色の矩形**: カメラ境界範囲
- **黄色の矩形**: デッドゾーン範囲

### 使用手順

1. Main Camera に `CameraFollow` を追加
2. `autoFindPlayer = true` のままなら自動でプレイヤーを追従
3. 必要に応じて `useBoundary` で移動範囲を制限

## MapTransition.cs

### クラス概要

プレイヤーが特定エリアに入ると別の場所（同一シーン内テレポート or シーン遷移）へ移動させる。

### 遷移タイプ

| タイプ | 説明 |
|-------|------|
| `Teleport` | 同一シーン内でテレポート（高速、ロードなし） |
| `SceneChange` | 別シーンへ遷移（Build Settings に登録が必要） |

### Inspector参照

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `transitionType` | TransitionType | 遷移タイプ |
| `teleportDestination` | Transform | テレポート先（Teleport時） |
| `arrivalDirection` | Vector2 | 到着後のプレイヤーの向き |
| `targetSceneName` | string | 遷移先シーン名（SceneChange時） |
| `spawnPointName` | string | 遷移先のスポーン位置名 |
| `useFade` | bool | フェードを使用するか（デフォルト: true） |
| `fadeDuration` | float | フェード時間（0.1〜2秒、デフォルト: 0.3） |
| `transitionSound` | AudioClip | 遷移時のSE（オプション） |

### 動作仕様

#### Teleport（同一シーン内）
1. プレイヤーがトリガーに入る
2. プレイヤーの移動を禁止
3. フェードアウト（`useFade = true` の場合）
4. プレイヤーとカメラを `teleportDestination` に移動
5. プレイヤーの向きを `arrivalDirection` に設定
6. フェードイン
7. プレイヤーの移動を許可

#### SceneChange（シーン遷移）
1. プレイヤーがトリガーに入る
2. プレイヤーの移動を禁止
3. フェードアウト
4. `nextSpawnPointName` と `nextArrivalDirection` を静的変数に保存
5. `SceneManager.LoadScene()` で遷移
6. 遷移先シーンで `spawnPointName` と同名の MapTransition がプレイヤーを配置

### Gizmo表示

- **水色の矩形**: Teleport の遷移ポイント
- **マゼンタの矩形**: SceneChange の遷移ポイント
- **黄色の線**: テレポート先への線
- **緑色の矢印**: 到着後の向き

### 必須コンポーネント

- `Collider2D`（自動追加、Is Trigger = true に設定必須）

### 使用手順（Teleport）

1. 遷移元に空の GameObject を作成
2. `BoxCollider2D` を追加（Is Trigger = ON）
3. `MapTransition` を追加
4. `transitionType = Teleport` を選択
5. 遷移先に空の GameObject を作成（目印用）
6. `teleportDestination` に遷移先をドラッグ

### 使用手順（SceneChange）

1. 遷移元シーンに遷移ポイントを作成（上記と同様）
2. `transitionType = SceneChange` を選択
3. `targetSceneName` に遷移先シーン名を入力
4. `spawnPointName` にスポーン位置名を入力
5. 遷移先シーンに同名の GameObject（MapTransition付き）を作成

## FadeManager.cs

### クラス概要

画面のフェードイン・フェードアウトを管理するシングルトン。シーンをまたいで動作する。

### Inspector参照

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `fadeImage` | Image | 画面全体を覆う Image |
| `fadeColor` | Color | フェードの色（デフォルト: 黒） |
| `defaultDuration` | float | デフォルトのフェード時間（秒） |
| `fadeInOnStart` | bool | ゲーム開始時にフェードインするか |

### 主要プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `Instance` | FadeManager | シングルトンインスタンス（静的） |
| `IsFading` | bool | 現在フェード中かどうか |

### 主要メソッド

| メソッド | 説明 |
|---------|------|
| `FadeOut(float duration)` | 画面を暗くする |
| `FadeIn(float duration)` | 画面を明るくする |
| `StartFade(float targetAlpha, float duration)` | 指定の透明度にフェード |
| `SetFadeOutImmediate()` | 即座に暗転 |
| `SetFadeInImmediate()` | 即座に明転 |
| `FadeOutAndIn(float out, float wait, float in)` | フェードアウト→待機→フェードイン |
| `SetFadeColor(Color color)` | フェード色を変更 |

### 動作仕様

1. **シングルトン**: `Awake` で `Instance` を設定、重複時は破棄
2. **DontDestroyOnLoad**: シーン遷移しても破棄されない
3. **コルーチン制御**: 新しいフェード開始時に既存のフェードを中断
4. **開始時フェードイン**: `fadeInOnStart = true` ならゲーム開始時に暗転→明転

### セットアップ手順

1. Canvas を作成（Screen Space - Overlay）
2. Canvas 直下に Image を作成
   - Anchor: Stretch（Alt+Shift+クリックで全画面）
   - Color: 黒（RGBA: 0, 0, 0, 255）
   - Raycast Target: OFF
3. Canvas に `FadeManager` を追加
4. `fadeImage` に Image をアサイン

### 注意事項

- `DontDestroyOnLoad` により Canvas がシーン遷移後も残る
- 新しいシーンの Canvas と重複しないよう設計に注意
- フェード中は `IsFading` プロパティで確認可能

## Input System 連携

### 使用アクション

| アクションマップ | アクション | キーバインド |
|----------------|-----------|-------------|
| `Player` | `Move` | WASD / 矢印キー / 左スティック |
| `Player` | `Interact` | E キー（長押し）/ Y ボタン |

### 注意点

- `Interact` アクションには `Hold` インタラクションが設定されている
- 即時反応が必要な場合は `Hold` を削除
